PAGE STRUCTURE — MUSSON BLOG
Описание всех пользовательских страниц, их назначения, структуры и поведения. Используется для проектирования фронтенда и реализации pixel-perfect интерфейса в соответствии с референсом Read WP.
ГЛАВНАЯ СТРАНИЦА

URL: / Назначение: список последних постов по дате публикации (10 на страницу)
Блоки:
Header (общий для всех страниц)
Content: • Заголовок страницы: «Последние статьи» • Список постов: заголовок, превью (обложка), анонс (краткое описание), теги, рейтинг, дата публикации • Пагинация: в самом низу блока
Footer (общий для всех страниц)
SEO:

: заполняется автоматически на основе первых параграфов постов, но обязательно доступно для ручного редактирования через Django Admin

СТРАНИЦА ПОСТА

URL: /posts// Назначение: отображение полной статьи
Блоки:
Header
Content: • Заголовок (H1) — post.title • Обложка (если есть) • Под заголовком в одну строку: теги (со ссылками), рейтинг (средняя оценка + голосование), дата публикации • Рядом — иконка шаринга с выпадающим меню: ВКонтакте, Telegram, Скопировать короткую ссылку • Тело поста
Footer
SEO:

: из search_description или извлекается автоматически из контента. Ручное редактирование через Django Admin обязательно.

OpenGraph, Twitter Card (заголовок, описание, обложка)
СТРАНИЦА ТЕГА

URL: /tags// Назначение: показать все посты с определённым тегом
Блоки:
Header
Content: • Заголовок страницы: «Теги» • Список всех тегов: в скобках количество постов с этим тегом, кликабельно • При переходе по тегу: заголовок «Все посты с тегом: #тег», список постов как на главной • Пагинация для страницы тега
Footer
SEO:

: генерируется автоматически или редактируется вручную через Django Admin

СТРАНИЦА АРХИВА

URL: /archive/ Назначение: многоуровневая навигация по дате публикации постов
Блоки:
Header
Content: • Список годов (например: 2023, 2024, 2025), рядом — количество постов за год • При переходе по ссылке /archive/2025/ — список месяцев (например: /archive/2025/april/), с количеством постов в каждом • При переходе по ссылке /archive/2025/april/ — список дат месяца (например: /archive/2025/april/25/), с количеством постов за каждую дату • При переходе на дату — список постов, как на главной странице, с пагинацией
Footer
SEO:

: генерируется автоматически

СТРАНИЦА ПОИСКА

URL: /search/?q=<строка> Назначение: поиск по заголовкам и содержимому, с учётом опечаток
Блоки:
Header + строка поиска
Content: • «Результаты по запросу: …» • Список постов (если есть), как на главной • Если ничего не найдено: сообщение «Ничего не найдено», а ниже — блок с рекомендациями похожих постов (на основе совпадений по тегам, заголовкам или ключевым словам)
Footer
SEO:
(страница не индексируется)
ОБО МНЕ

URL: /about/ Назначение: авторская страница с кратким представлением
Блоки:
Header
Content: • Фото автора, краткое био, ссылки на внешние ресурсы (Telegram, GitHub и т.п.)
Footer
SEO:

: редактируется через админку
КОНТАКТЫ

URL: /contact/ Назначение: предоставление формы обратной связи с защитой от спама
Блоки:
Header
Content: • Краткий текст • Форма обратной связи с капчей от ботов • Контактные ссылки не отображаются на этой странице
Footer
SEO:

: редактируемое поле через админку
СТРАНИЦА КОРОТКОЙ ССЫЛКИ (Redirect)

URL: /s/id/ Назначение: редирект на полный URL поста
Поведение:
Поиск ShortLink.code → редирект на post.get_absolute_url()
При отсутствии — 404
SEO:
(редирект не индексируется)
